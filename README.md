# Secure Coding Review – Task 3

## Cybersecurity Internship

This repository contains **Task 3: Secure Coding Review**, completed as part of a cybersecurity internship.  
The task focuses on reviewing a simple **C++ login application** to identify common security vulnerabilities and recommend secure coding practices.

---

## 🔹 Programming Language
- C++

## 🔹 Application Reviewed
- Simple Login System

---

## 🔹 Task Objective
The objective of this task is to analyze source code through **manual inspection** and identify security issues such as:
- Hardcoded credentials  
- Plain text password handling  
- Lack of input validation  

Based on the findings, secure coding **recommendations and best practices** are suggested to improve application security.

---

## 🔹 Contents of This Repository
- Secure Coding Review document (DOC / PDF)
- Sample vulnerable C++ code
- Identified security vulnerabilities
- Secure coding recommendations
- Conclusion

---

## 🔹 Key Learnings
- Importance of secure authentication mechanisms  
- Risks of hardcoded credentials  
- Role of secure coding reviews in preventing cyber attacks  

---

## 🔹 Conclusion
Secure coding reviews help developers build **safe, reliable, and secure applications**. Regular code reviews reduce security risks and improve overall software quality.
